# 프로젝트 구조 문서

> 코드 구조를 문서로 관리하여 AI가 코드를 직접 읽지 않고도 구조 파악 가능
> **컨텍스트 절약**: 코드보다 이 문서를 먼저 참조

---

## 컨텍스트 절약 규칙

```
✅ 단순 구조 확인 (Entity 필드, API 엔드포인트, DB 스키마 등)
   → 실제 소스 코드를 읽기 전에 이 폴더 내부 문서를 먼저 참조

❌ 복잡한 로직 분석, 버그 수정
   → 실제 코드 읽기 필요
```

---

## 폴더 구조

```
docs/structure/
├── README.md                 # 이 파일
├── backend/
│   ├── {domain}/             # 도메인별 폴더
│   │   ├── api.md            # API 엔드포인트
│   │   └── db.md             # DB 스키마
│   └── common/
│       └── overview.md       # 공통 컴포넌트
└── frontend/
    ├── api.md                # API 클라이언트
    └── pages.md              # 페이지 구성
```

---

## 문서 목록

### Backend

| 문서 | 설명 | 언제 참조? |
|------|------|-----------|
| `backend/{domain}/api.md` | API 엔드포인트, 요청/응답 | API 개발, 연동 시 |
| `backend/{domain}/db.md` | DB 스키마, 엔티티 관계 | Entity 작성, 쿼리 작성 시 |
| `backend/common/overview.md` | 공통 컴포넌트, 예외 처리 | 공통 기능 파악 시 |

### Frontend

| 문서 | 설명 | 언제 참조? |
|------|------|-----------|
| `frontend/api.md` | API 클라이언트 함수 | API 연동 시 |
| `frontend/pages.md` | 페이지 구성, 라우팅 | 페이지 개발 시 |

## 도메인 다이어그램 (예시)

```
┌─────────────────────────────────────────────────────────────┐
│                         User                                 │
│                    (사용자 - 최상위 단위)                     │
├─────────────────────────────────────────────────────────────┤
│  id, email, password, name, role, status                    │
└──────────────────────┬──────────────────────────────────────┘
                       │ 1:N
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                       Course                                 │
│                      (강의)                                  │
├─────────────────────────────────────────────────────────────┤
│  id, title, description, instructor_id(FK)                  │
└──────────────────────┬──────────────────────────────────────┘
                       │ 1:N
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     Enrollment                               │
│                    (수강 신청)                               │
├─────────────────────────────────────────────────────────────┤
│  id, user_id(FK), course_id(FK), status                     │
└─────────────────────────────────────────────────────────────┘
```

> **Note**: 위 다이어그램은 예시입니다. 실제 프로젝트에 맞게 수정하세요.
