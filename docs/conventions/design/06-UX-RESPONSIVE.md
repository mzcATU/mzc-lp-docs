# 06. UX Responsive

> 반응형 브레이크포인트별 레이아웃 변환 규칙

---

## 핵심 규칙

```
✅ 모바일 우선 설계 (Mobile First)
✅ 브레이크포인트별 레이아웃 전환 규칙 준수
✅ 터치 타겟 최소 44x44px (모바일)
✅ 콘텐츠 우선순위에 따른 표시/숨김
```

---

## 1. 브레이크포인트 정의

### 1.1 Tailwind 브레이크포인트

| 이름 | 크기 | 대상 | 사용 예시 |
|------|------|------|-----------|
| (기본) | < 640px | 모바일 | 기본 스타일 |
| `sm` | ≥ 640px | 모바일 가로 | `sm:flex` |
| `md` | ≥ 768px | 태블릿 | `md:grid-cols-2` |
| `lg` | ≥ 1024px | 데스크톱 | `lg:px-8` |
| `xl` | ≥ 1280px | 대형 화면 | `xl:max-w-7xl` |
| `2xl` | ≥ 1536px | 초대형 화면 | `2xl:grid-cols-4` |

### 1.2 디바이스별 주요 브레이크포인트

```
┌─────────────────────────────────────────────────────────────┐
│  Mobile        Tablet         Desktop        Large         │
│  < 768px       768-1023px     1024-1279px    ≥ 1280px      │
│                                                             │
│  ├────────────┼─────────────┼─────────────┼─────────────┤  │
│  0           768           1024          1280          ∞   │
│              (md)          (lg)          (xl)              │
└─────────────────────────────────────────────────────────────┘
```

## 2. 레이아웃 변환 규칙

### 2.1 전체 레이아웃

#### 모바일 (< 768px)

```
┌─────────────────────────────────────────┐
│  [Header]                        [☰]   │
├─────────────────────────────────────────┤
│                                         │
│            [Main Content]               │
│                                         │
├─────────────────────────────────────────┤
│  [🏠] [📚] [👤] [⚙️]                    │
│  (Bottom Navigation)                    │
└─────────────────────────────────────────┘
```

#### 태블릿 (768px - 1023px)

```
┌─────────────────────────────────────────┐
│  [Header]                    [User] [🔔]│
├──────────┬──────────────────────────────┤
│  [Mini   │                              │
│  Sidebar]│       [Main Content]         │
│  (Icons) │                              │
│          │                              │
└──────────┴──────────────────────────────┘
```

#### 데스크톱 (≥ 1024px)

```
┌─────────────────────────────────────────────────────────────┐
│  [Header]                                        [User] [🔔]│
├────────────────┬────────────────────────────────────────────┤
│                │                                            │
│  [Sidebar]     │           [Main Content]                   │
│  (Full Width)  │                                            │
│                │                                            │
│                │                                            │
└────────────────┴────────────────────────────────────────────┘
```

### 2.2 사이드바 변환

| 브레이크포인트 | 사이드바 상태 | 너비 |
|---------------|--------------|------|
| 모바일 (< 768px) | 숨김 → 하단 네비게이션 | 0px |
| 태블릿 (md) | 아이콘만 표시 | 64px |
| 데스크톱 (lg) | 전체 표시 (접기 가능) | 240px / 64px |

```tsx
// 사이드바 반응형
<aside className="
  hidden md:flex           // 모바일 숨김, 태블릿 이상 표시
  w-16 lg:w-60             // 태블릿: 64px, 데스크톱: 240px
  transition-all duration-200
">
  {/* 사이드바 콘텐츠 */}
</aside>

// 하단 네비게이션 (모바일 전용)
<nav className="
  md:hidden                // 태블릿 이상에서 숨김
  fixed bottom-0 left-0 right-0
  h-16 bg-white border-t
">
  {/* 네비게이션 아이템 */}
</nav>
```

### 2.3 그리드 변환

#### 카드 그리드

```
┌─────────────────────────────────────────────────────────────┐
│  [Mobile: 1 column]                                         │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Card 1                                                 ││
│  └─────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Card 2                                                 ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  [Tablet: 2 columns]                                        │
│  ┌────────────────────────┐  ┌────────────────────────┐    │
│  │  Card 1                │  │  Card 2                │    │
│  └────────────────────────┘  └────────────────────────┘    │
│  ┌────────────────────────┐  ┌────────────────────────┐    │
│  │  Card 3                │  │  Card 4                │    │
│  └────────────────────────┘  └────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  [Desktop: 3-4 columns]                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │  Card 1  │  │  Card 2  │  │  Card 3  │  │  Card 4  │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
└─────────────────────────────────────────────────────────────┘
```

```tsx
<div className="
  grid
  grid-cols-1           // 모바일: 1열
  sm:grid-cols-2        // 태블릿: 2열
  lg:grid-cols-3        // 데스크톱: 3열
  xl:grid-cols-4        // 대형 화면: 4열
  gap-4 md:gap-6
">
  {items.map(item => <Card key={item.id} />)}
</div>
```

### 2.4 테이블 → 카드 변환

#### 데스크톱: 테이블

```
┌─────────────────────────────────────────────────────────────┐
│  강의명           │ 차시  │ 수강생 │ 상태   │ 생성일        │
├─────────────────────────────────────────────────────────────┤
│  React 기초       │  12   │   45   │ 진행중 │ 2024-12-01   │
│  TypeScript 심화  │   8   │   32   │ 준비중 │ 2024-12-05   │
└─────────────────────────────────────────────────────────────┘
```

#### 모바일: 카드

```
┌─────────────────────────────────────────┐
│  React 기초                             │
│  ─────────────────────────────────────│
│  차시: 12  │  수강생: 45                │
│  상태: 진행중  │  생성일: 2024-12-01    │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  TypeScript 심화                        │
│  ─────────────────────────────────────│
│  차시: 8   │  수강생: 32                │
│  상태: 준비중  │  생성일: 2024-12-05    │
└─────────────────────────────────────────┘
```

```tsx
{/* 데스크톱: 테이블 */}
<table className="hidden md:table w-full">
  <thead>...</thead>
  <tbody>
    {data.map(row => <tr key={row.id}>...</tr>)}
  </tbody>
</table>

{/* 모바일: 카드 */}
<div className="md:hidden space-y-4">
  {data.map(row => (
    <div key={row.id} className="p-4 border rounded-lg">
      <h3 className="font-medium">{row.title}</h3>
      <div className="mt-2 text-sm text-text-secondary">
        <span>차시: {row.count}</span>
        <span>수강생: {row.students}</span>
      </div>
    </div>
  ))}
</div>
```

## 3. 반응형 타이포그래피

| 요소 | 모바일 | 태블릿 | 데스크톱 |
|------|--------|--------|----------|
| H1 (페이지 제목) | `text-xl` (20px) | `text-2xl` (24px) | `text-3xl` (30px) |
| H2 (섹션 제목) | `text-lg` (18px) | `text-xl` (20px) | `text-2xl` (24px) |
| Body | `text-sm` (14px) | `text-base` (16px) | `text-base` (16px) |
| Caption | `text-xs` (12px) | `text-sm` (14px) | `text-sm` (14px) |

```tsx
<h1 className="text-xl md:text-2xl lg:text-3xl font-bold">
  페이지 제목
</h1>
```

## 4. 반응형 간격

| 요소 | 모바일 | 태블릿 | 데스크톱 |
|------|--------|--------|----------|
| 페이지 패딩 | `p-4` (16px) | `p-6` (24px) | `p-8` (32px) |
| 섹션 마진 | `my-6` (24px) | `my-8` (32px) | `my-12` (48px) |
| 카드 갭 | `gap-4` (16px) | `gap-6` (24px) | `gap-6` (24px) |

```tsx
<div className="p-4 md:p-6 lg:p-8">
  <section className="my-6 md:my-8 lg:my-12">
    ...
  </section>
</div>
```

## 5. 터치 타겟

### 최소 크기

| 요소 | 최소 크기 | Tailwind |
|------|----------|----------|
| 버튼 | 44 × 44px | `min-h-11 min-w-11` |
| 아이콘 버튼 | 44 × 44px | `h-11 w-11` |
| 링크 영역 | 44px 높이 | `py-3` |
| 체크박스/라디오 | 44 × 44px (터치 영역) | 패딩으로 확장 |

```tsx
// 모바일에서 터치 타겟 확대
<button className="
  h-10 md:h-9           // 데스크톱에서 약간 작게
  min-w-[44px]          // 최소 터치 영역
  px-4
">
  버튼
</button>

// 아이콘 버튼
<button className="
  h-11 w-11 md:h-9 md:w-9
  flex items-center justify-center
">
  <Icon className="h-5 w-5" />
</button>
```

## 6. 콘텐츠 우선순위

### 표시/숨김 규칙

| 콘텐츠 | 모바일 | 태블릿 | 데스크톱 |
|--------|--------|--------|----------|
| 필수 정보 | ✅ | ✅ | ✅ |
| 보조 정보 | 접기/더보기 | ✅ | ✅ |
| 부가 정보 | ❌ | 축소 | ✅ |

```tsx
{/* 필수: 항상 표시 */}
<div>필수 정보</div>

{/* 보조: 모바일에서 더보기 */}
<div className="hidden sm:block">보조 정보</div>
<button className="sm:hidden">더보기</button>

{/* 부가: 데스크톱만 */}
<div className="hidden lg:block">부가 정보</div>
```

- [ ] 모바일 우선 설계 (기본 스타일 = 모바일)

- [ ] 테이블 → 카드 전환
- [ ] 폼 스택 → 그리드 전환
- [ ] 타이포그래피 스케일 확인
- [ ] 모든 브레이크포인트에서 테스트
